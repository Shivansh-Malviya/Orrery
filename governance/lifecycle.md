# Governance Lifecycle

## Decision flow

```
Proposal → Review → Decision → Record → Retrospective
```

1. **Proposal** — Anyone proposes a governance change (via issue, PR, or `.context/promotion_queue.md`)
2. **Review** — Governance reviewer evaluates against principles, existing policy, and impact
3. **Decision** — Approve, reject, defer, or refer
4. **Record** — Decision recorded in `governance/context/decision-log.md`
5. **Retrospective** — After implementation, lessons recorded in `governance/learner/`

## Staleness check

Governance documents older than 90 days without review are flagged for refresh.
