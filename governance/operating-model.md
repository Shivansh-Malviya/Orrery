# Governance Operating Model

## Scope

Governance covers:
- Structural changes to Orrery root directories
- Changes to `.safe/locked/` documents
- Addition/removal of canonical sources
- Cross-cutting policy changes

## Out of scope

- Day-to-day project work in workspace modules
- Content within `.context/` files
- Routine `.agents/rules/` updates

## Model

Governance operates as a lightweight reviewer-in-loop, not a gate.
The default assumption is yes; governance exists to catch unforced errors.
