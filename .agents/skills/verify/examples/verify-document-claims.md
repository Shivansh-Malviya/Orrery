# Example — Verify Document Claims

## Claim ledger

| ID | Claim | Expected Evidence |
|---|---|---|
| C1 | A table includes all eligible exams. | Eligibility criteria from authoritative sources and row inventory. |
| C2 | Dates are current. | Current official notification/calendar pages with retrieval dates. |
| C3 | No ineligible exams are included. | Negative check against age/degree/category constraints. |

## Checks

- Extract every factual claim.
- Verify dates from current authoritative sources.
- Cross-check eligibility against user constraints.
- Search document for stale or contradictory entries.
