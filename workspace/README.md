# Workspace

Federated workspaces for product OSes, domains, routing areas, and project namespaces.

## Workspace Types

A workspace entry may be:

1. **Product OS**: `lattice/`, `argos/`
2. **Restricted shim**: `work/`
3. **Domain workspace**: `research/`, `education/`, `me/`
4. **Routing workspace**: `inbox/`
5. **Project namespace**: `side-projects/`
6. **Compatibility bridge**: `career/`

Do not keep a generic domain folder as an active authority when a product OS owns that domain.
